# [Silver II] N번째 큰 수 - 2075 

[문제 링크](https://www.acmicpc.net/problem/2075) 

### 성능 요약

메모리: 2156 KB, 시간: 368 ms

### 분류

자료 구조, 우선순위 큐, 정렬

### 아이디어

- 모든 숫자를 우선순위 큐에 저장한 후 N번째 큰 값까지 pop해서 N번째 큰 수 찾기 : 메모리 초과
- 메모리 관리하기 위해서 모든 숫자를 우선순위 큐에 저장하는 것이 아닌 N개 숫자만 저장한다.
  - 저장된 숫자 개수가 N개를 초과한다면 가장 작은 숫자를 제거 (pq.pop())하면서 N개 숫자만 우선순위 큐에서 관리되도록 유지
